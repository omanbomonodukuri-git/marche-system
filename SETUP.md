# 🔆 レーザー工房 マルシェ管理システム セットアップ手順

---

## 必要なもの
- Google アカウント
- GitHub アカウント（お持ちとのこと）

---

## STEP 1：Googleスプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 「空白のスプレッドシート」を新規作成
3. スプレッドシートのURLからIDをコピー

   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```

---

## STEP 2：Google Apps Script を設定

1. スプレッドシートのメニュー → **拡張機能 → Apps Script**
2. 左のファイル一覧の「コード.gs」を全選択して削除
3. `gas_code.gs` の内容を全部貼り付け
4. ファイル上部の `SPREADSHEET_ID` を変更：

   ```javascript
   const SPREADSHEET_ID = '先ほどコピーしたID';
   ```

5. **保存**（Ctrl+S または ⌘+S）

---

## STEP 3：初期セットアップを実行

1. Apps Script エディタで関数一覧から **`setup`** を選択
2. ▶ 実行ボタンをクリック
3. 「承認が必要です」が出たら → 「許可を確認」→ Googleアカウントを選択 → 「詳細」→ 「（プロジェクト名）に移動」→ 「許可」
4. ログに `✅ セットアップ完了！` と表示されればOK

> デフォルトパスワード：**`marche2024`**（後で変更推奨）

---

## STEP 4：ウェブアプリとしてデプロイ

1. Apps Script エディタ右上の「**デプロイ**」→「**新しいデプロイ**」
2. 種類の選択（⚙️歯車アイコン）→ **ウェブアプリ**
3. 設定：
   - 説明：`マルシェ管理API`
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**（※ パスワード保護はアプリ側で行います）
4. 「デプロイ」→ **ウェブアプリのURL** をコピー

---

## STEP 5：HTMLファイルにURLを設定

`index.html` をテキストエディタで開き、以下を変更：

```javascript
const GAS_URL = 'ここにGASデプロイURLを貼り付け';
```

→ STEP 4 でコピーしたURLを貼り付け

---

## STEP 6：GitHub Pages に公開

1. GitHub で新しいリポジトリを作成（例：`marche-system`）
   - **Private** にする（公開したくない場合）
   - ただしGitHub PagesはPrivateの場合、無料プランでは使えないため **Public** を推奨
     （パスワード保護はアプリ内で行います）

2. `index.html` をリポジトリにアップロード

3. リポジトリの **Settings → Pages**
   - Source: **Deploy from a branch**
   - Branch: **main** / **/(root)**
   - Save

4. 数分後に公開URL（`https://【ユーザー名】.github.io/marche-system/`）が発行される

---

## STEP 7：動作確認

1. 公開URLをiPadのSafariで開く
2. パスワード `marche2024` でログイン
3. 商品マスターにキーホルダーなどを登録
4. 新規注文を受け付けてみる

---

## パスワードを変更する場合

1. Google Apps Script エディタを開く
2. `setup()` 関数内の以下を変更：
   ```javascript
   const defaultPassword = 'marche2024'; // ← 任意のパスワードに変更
   ```
3. `setup()` を再実行（シートが既にある場合は新しいhashだけ上書きされます）

---

## データの確認・エクスポート

- アプリ内「📊 売上」タブ →「Googleスプレッドシートで開く」ボタン
- スプレッドシートを直接開いて確認・CSV出力も可能
  - ファイル → ダウンロード → CSV形式

---

## シートの構成

| シート名 | 内容 |
|---------|------|
| `orders` | 注文一覧（受付番号・お渡し方法・メモ等） |
| `items` | 注文内商品明細（商品・個数・価格・支払方法） |
| `steps` | 各行程の進捗・開始時刻・完了時刻・実績時間 |
| `products` | 商品マスター（行程ごとの作業時間含む） |
| `history` | 完了済み注文（待ち時間実績） |
| `sales` | 売上明細（支払方法・金額別） |
| `config` | パスワードハッシュ等 |

---

## トラブルシューティング

**「通信中…」が消えない**
→ GAS URLが正しく設定されているか確認。Apps Scriptのデプロイを再実行。

**ログインできない**
→ `setup()` を再実行してパスワードをリセット。

**2台のiPadでデータが異なる**
→ ページをリロード（引っ張って更新）。GASが自動同期の役割を果たします。

## 使い方クイックガイド

### 美咲さん（受付・デザイン・撮影・完成連絡）
1. お客様から注文 → 「＋新規注文」→ 受付番号・商品・個数・お渡し方法を選択
2. 同じ写真・デザインの商品は「💰 価格・支払」から「スキップ」設定
3. 各行程のボタンをタップして進捗を更新

### 栄人さん（2値化・彫刻）
1. 自分の担当行程（🖼 2値化、🔆 彫刻）が光ったらタップして完了
2. スキップ設定済みの行程は自動でグレーアウト

### 待ち時間の仕組み
- 行程の**完了チェックが入るまで**は残り時間から引かれません
- 当日渡しのみの待ち時間を計算・表示
- お渡し目安時刻（例：14:30）も各注文に表示

## GAS 自動デプロイ セットアップ（GitHub Actions）

コードを GitHub にプッシュすると **GAS も自動でデプロイ**されます。
初回のみ以下の手順でセットアップしてください。

### 必要なもの
- パソコン（macOS/Windows）に [Node.js](https://nodejs.org/) をインストール

### 手順

**1. clasp をインストール・ログイン**
```bash
npm install -g @google/clasp
clasp login
```
ブラウザでGoogleアカウントの認証画面が開くので許可します。

**2. credentials ファイルを確認**
```bash
cat ~/.clasprc.json
```
↑ 表示された JSON 全体をコピーします（長い文字列）

**3. GAS スクリプトIDを確認**
Apps Script エディタを開く → 左メニュー「⚙️ プロジェクトの設定」→
「スクリプト ID」をコピー

**4. GitHub に Secrets を登録**
GitHub リポジトリ → Settings → Secrets and variables → Actions → New repository secret

| Secret 名 | 値 |
|-----------|---|
| `CLASP_CREDENTIALS` | 手順2でコピーした JSON |
| `GAS_SCRIPT_ID` | 手順3でコピーしたスクリプトID |

**5. 動作確認**
`src/gas_code.gs` を変更して `git push` すると、GitHub Actions が自動でGASにデプロイします。
リポジトリの「Actions」タブで進捗を確認できます。
